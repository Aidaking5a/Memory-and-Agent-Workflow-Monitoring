# Security Policy

## Supported Versions

This repository is currently in active pre-release development.

## Reporting a Vulnerability

Please report security issues privately to: `security@theia.example`.

Include:
- Affected package or app
- Reproduction details
- Potential impact
- Suggested mitigation (optional)

Do not open public issues for exploitable vulnerabilities.

## Security Commitments

- Least-privilege access model
- No covert collection patterns
- Signed release artifacts for distribution builds
- Dependency and secret scanning in CI
- Auditability for permission and policy actions

## Non-Goals

Theia is not designed for unauthorized access, stealth monitoring, keylogging, or offensive cyber behavior.
