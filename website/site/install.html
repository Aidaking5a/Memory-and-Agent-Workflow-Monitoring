Install Configurator

Pick your platform and setup goal. Theia gives you one exact command and keeps advanced connector paths out of the homepage so customers do not have to guess.

Safe By Default

  • Scripts are downloaded to a temp file and printed before execution.
  • Node, pnpm, Git, Docker, WSL, Python, paid APIs, and cloud keys are never silently installed.
  • Cloud and OAuth connector credentials stay opt-in and local to your setup.

Build Your Command

Use these controls when you need a specific customer setup, local dev run, or connector validation path.

Windows Clean Install

Downloads the Windows installer script, prints it for review, builds the dashboard, and starts Theia.

$ powershell -NoProfile -ExecutionPolicy Bypass -Command "$u='https://raw.githubusercontent.com/aidaking5a/Memory-and-Agent-Workflow-Monitoring/main/scripts/install-theia-command-center.ps1'; $p=Join-Path $env:TEMP 'install-theia-command-center.ps1'; Invoke-WebRequest $u -OutFile $p; Get-Content $p; powershell -NoProfile -ExecutionPolicy Bypass -File $p -BuildDashboard -StartAfterInstall"

After It Runs

  1. Open the dashboard at http://localhost:5173.
  2. Use Connect Agent to pair OpenClaw, Octopoda, MCP, API/OAuth, terminal, or custom JSON reporters.
  3. Validate the connector, then confirm live bubbles, stats, and agent cards update.

Recommended Customer Path

Use Clean Install on Windows first. It is the clearest path for non-technical users and keeps every risky or paid dependency explicit.

OpenClaw Path

The default Windows dashboard command searches C:\Users\admin_1\src\openclaw. The macOS path mirrors that as $HOME/src/openclaw.

Connector Rule

Install commands start Theia. Connector keys, OAuth grants, Octopoda, MCP clients, and paid APIs are paired manually from the dashboard.
