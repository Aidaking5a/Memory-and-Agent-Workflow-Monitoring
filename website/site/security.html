Security And Trust Center

Theia is built for authorized oversight only. No covert monitoring, no credential theft patterns, and no privilege bypass behavior.

Permission Boundaries

  • Connector and file scopes are explicit and revocable
  • Least-privilege defaults for each workspace
  • No hidden access collection

Local-First Processing

  • Memory parsing and reasoning analysis run locally by default
  • Optional cloud control-plane is opt-in
  • Sensitive fields can be redacted pre-export

Identity And Access

  • SAML-ready control-plane authentication
  • Role-based policy model (owner, operator, reviewer, auditor)
  • Session audit events for login and logout traces

Auditability

  • Tamper-evident chain hash model
  • Permission grant/revoke event history
  • Run timeline evidence references

Operational Controls

  • Signed release workflow (recommended for distribution)
  • CI quality gates for public repository hygiene
  • Security disclosure channel via SECURITY.md

Data Lifecycle

  • Explicit offboarding and deletion paths
  • Transparent retention behavior
  • No hidden telemetry monetization
